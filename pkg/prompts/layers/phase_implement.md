# Phase: Implement

The goal of this phase is to **write the code changes** according to the plan.

## What to do

1. Follow the plan from the previous phase
2. Make the necessary file changes
3. Ensure changes match existing code patterns
4. Add appropriate comments and documentation

## Tools available

You have access to: file, code_edit, search, navigate, bash, git

## Output

Return a JSON object with:
- `files_modified`: Array of files that were changed
- `summary`: Brief description of what was implemented
- `tests_added`: Any new tests added (if applicable)
