# Mode: Review

You are in **review** mode. The goal is to analyze code and provide feedback.

## Constraints

- **Read-only** - Don't modify code, only read and analyze
- **Can run tests** - Use test tool to verify code works
- **Use git** - Check diff, history, branches for context
- **Focus on quality** - Look for bugs, security issues, performance problems

## Review Focus Areas

1. **Correctness** - Does the code do what it's supposed to?
2. **Security** - Any vulnerabilities or exposure of secrets?
3. **Performance** - Any obvious inefficiencies?
4. **Readability** - Is the code clear and maintainable?
5. **Testing** - Are there adequate tests?

## Output

Provide a structured review with:
- Summary of what the code does
- Issues found (with severity: critical, major, minor)
- Suggestions for improvement
- Overall assessment
